Fluid Scaling

Every size — type, spacing, widths, heights — is set from two values: its size at a 360 px screen and its size at a 1440 px screen. The build renders those two exactly and scales continuously between and beyond them. This reference records how that behaves in every case.

01 The two anchors

360 px and 1440 px render the design values exactly. Screen widths between the two are interpolated. Widths above 1440 px zoom up to a maximum at 1920 px, then hold. A size is written [value @360, value @1440].

02 Three zones

Mobile360–480

The size zooms up with the screen — capped so it grows at most 1.33× its 360 px value, and never past its 1440 px value.

Glide480–1440

A straight line to the 1440 px value. Continuous with the mobile zone — no step at any breakpoint.

Wide1440–1920

A gentle zoom, reaching +16.7% at 1920 px. Held constant beyond 1920 px.

Horizontal: screen width (px). Vertical: rendered size (px). Dashed verticals mark the 360 and 1440 anchors; gold dots sit on the exact design values.

03 The cap

On mobile a size can grow to at most 1.33× its 360 px value, and never beyond its 1440 px value — whichever is smaller. This produces two behaviors, decided by how the 1440 px value compares to 1.33× the 360 px value:

Reach early, then hold

When the 1440 value is within 1.33× the 360 value, the size reaches it on mobile and holds flat across the whole desktop range. e.g. [12,14]

Glide all the way

When the 1440 value is larger than 1.33× the 360 value, the size hits the 1.33× ceiling on mobile, then keeps gliding to the 1440 value. e.g. [12,20]

04 Flat & frozen

Flat — equal values

When both values match [12,12], the size holds at that value from 360 through 1440 (there is nothing to grow toward), then takes the gentle wide-zone zoom above 1440. Used for text whose design size is the same on mobile and desktop.

Frozen — never moves

A separate marked form holds a value constant at every width, including the wide zone. Reserved for 1 px hairlines, borders, and alignment constants that must not scale at all.

05 Inverted sizes

The 360 value may be larger than the 1440 value [16,12]. The size then glides down from mobile to desktop, exact at both anchors, and holds beyond 1440. Used where an element is intentionally larger on phones than on desktop.

06 Single-anchor sizes

An element that exists on only one layout is defined by a single value rather than a pair.

Mobile-only

Defined by its phone value [24]. Zooms up with the screen to 1.33× (24 → 32 by 480 px), then holds. For elements shown only on phones.

Desktop-only

Defined by its desktop value [60]. Fixed at that value up to 1440 px, then the gentle wide-zone zoom (60 → 70 at 1920 px). For elements shown only on desktop.

07 Line-height & letter-spacing

Line-height

Set as a ratio of the font size, so it scales together with the text at every width — the line box stays proportional as the type grows or shrinks.

Letter-spacing

Set in em, so it also scales with the font. Where the design calls for it, letter-spacing can change to a different value at the desktop breakpoint (a step, not a glide).

09 Wide-screen page cap

Page content stops widening at 1920 px and centers on larger screens. Full-bleed color bands and imagery still reach the true screen edge; only the content column is capped.

10 Overlay text on banners

Headlines and captions set over banner imagery scale to the width of their copy column, not the viewport — so the text stays locked in proportion to the artwork it sits on as the banner resizes, and wrap points hold.

11 Accessibility

Browser & OS zoom

Sizes are rem-based, so increasing the browser or OS text size enlarges everything proportionally — the fluid scaling does not lock users out of zoom.

Form inputs

Text inside form fields stays at least 16 px on mobile, so focusing an input does not trigger the automatic page-zoom on iOS Safari.
